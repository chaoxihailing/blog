Executor与线程池

线程池是一种生产者-消费者模式，线程池的使用方是生产者，线程池本身是消费者

Java中线程池最核心的工具类，ThreadPoolExecutor, 里面构造成员，corePoolSize, maximumPoolSize,keepAliveTime & unit, workQueue,threadFactory,handler.

使用线程池要注意什么

不推荐使用静态工厂类Executors,因为提供很多方法都是无界的，高负载的时候，无界队列容易导致OOM.

FutureTask ft2

// 创建任务 T1 的 FutureTask
FutureTask ft1

Thread T1 = new Thread(ft1);
T1.start();
// 线程 T2 执行任务 ft2
Thread T2 = new Thread(ft2);
T2.start();
// 等待线程 T1 执行结果
System.out.println(ft1.get());

// T1Task 需要执行的任务：
// 洗水壶、烧开水、泡茶
class T1Task implements Callable{ FutureTask ft2;
// T1 任务需要 T2 任务的 FutureTask
T1Task(FutureTask ft2){
this.ft2 = ft2;
}
@Override
String call() throws Exception {
System.out.println(“T1: 洗水壶…”);
TimeUnit.SECONDS.sleep(1);

System.out.println("T1: 烧开水...");
TimeUnit.SECONDS.sleep(15);
// 获取 T2 线程的茶叶
String tf = ft2.get();
System.out.println("T1: 拿到茶叶:"+tf);

System.out.println("T1: 泡茶...");
return " 上茶:" + tf;

}
} // T2Task 需要执行的任务:
// 洗茶壶、洗茶杯、拿茶叶
class T2Task implements Callable {
@Override
String call() throws Exception {
System.out.println(“T2: 洗茶壶…”);
TimeUnit.SECONDS.sleep(1);

System.out.println("T2: 洗茶杯...");

} // 一次执行结果：

// 任务 1：洗水壶 -> 烧开水
CompletableFuture f1 =
CompletableFuture.runAsync(()->{
System.out.println(“T1: 洗水壶…”);
sleep(1, TimeUnit.SECONDS);

System.out.println(“T1: 烧开水…”);
sleep(15, TimeUnit.SECONDS);
});
// 任务 2：洗茶壶 -> 洗茶杯 -> 拿茶叶
CompletableFuture f2 =
CompletableFuture.supplyAsync(()->{
System.out.println(“T2: 洗茶壶…”);
sleep(1, TimeUnit.SECONDS);

System.out.println(“T2: 洗茶杯…”);
sleep(2, TimeUnit.SECONDS);

System.out.println(“T2: 拿茶叶…”);
sleep(1, TimeUnit.SECONDS);
return “ 龙井 “;
});
// 任务 3：任务 1 和任务 2 完成后执行：泡茶
CompletableFuture f3 =
f1.thenCombine(f2, (__, tf)->{
System.out.println(“T1: 拿到茶叶:” + tf);
System.out.println(“T1: 泡茶…”);
return “ 上茶:” + tf;
});
// 等待任务 3 执行结果
System.out.println(f3.join());

利用CompletionService实现Dubbo中的Forking Cluster

Dubbo有个Forking的集群模式，在这种集群模式下，支持并行地调用多个查询服务，只有有个一个成功返回结果，整个服务就可以返回了。

Fork/Join

Fork/Join 是一个并行计算的框架，主要用来支持分治模型的，fork对应的任务分解，Join对应的是结果合并

ForkJoinPool 本质上也是 生产者-消费者模型，但是更为智能，ThreadPoolExecutor内部只有一个任务队列，而ForkJoinPool内部有多个任务队列
当工作线程对应的任务队列空了，那么它会窃取其他工作任务的任务，ForkJoinPool的任务队列采用的双端队列。工作线程正常获取和窃取分别是从任务队列不同端消费。
